## 2. 技术栈选型规则 (Stack Selection Rules)

AI 必须按顺序匹配以下规则，命中即停，不得随意更改。

### 2.1 后端框架
1.  **数据科学/AI/爬虫：** 强制 `FastAPI` (Python)。
2.  **Python 偏好：** 高性能/实时选 `FastAPI`，否则 `Flask`。
3.  **JS/TS 偏好：** 高性能选 `Fastify`，否则 `Express`。
4.  **Go 偏好：** 强制 `Gin`。
5.  **其他：** Java(`Spring Boot`), PHP(`Laravel`), Ruby(`Rails`)。
6.  **默认：** `FastAPI`。

### 2.2 数据库与 ORM (若 database_needed=true)
1.  **Python:** 高性能 `SQLAlchemy`，异步 `Tortoise ORM`。
2.  **JS/TS:** 强制 `Prisma` (类型安全)。
3.  **Go:** 强制 `GORM`。
4.  **默认：** SQLite (开发) + PostgreSQL (生产)。

### 2.3 前端状态管理
1.  **简单应用：** 框架原生 (React Context / Vue Reactive)。
2.  **复杂应用：** `Zustand` (React) 或 `Pinia` (Vue)。
3.  **服务端状态：** 强制 `React Query` 或 `SWR` (管理 API 缓存与同步)。

## 4. API 契约与安全规范 (API Contract & Security)

### 4.1 统一响应格式
所有接口必须返回以下 JSON 结构：
```json
{
  "code": 0,          // 0=成功，非 0=业务错误
  "message": "success",
  "data": null        // 具体数据
}
```

### 4.2 HTTP 状态码规范
-  **成功：** HTTP 200 + 业务 code: 0。
-  **客户端错误：** HTTP 400/401/403/404 + 业务 code: 4xx。
-  **服务端错误：** HTTP 500 + 业务 code: 5xx (前端提示"系统繁忙")。
-  **前端拦截：** 必须封装 Axios 拦截器，统一处理 HTTP 错误与业务 code 错误。

### 4.3 安全强制项
-  **CORS：** 后端必须配置跨域中间件，仅允许指定域名。
-  **鉴权：** 所有写操作 (POST/PUT/DELETE) 必须验证 Token (JWT/Session)。
-  **输入验证：** 后端必须二次验证所有前端输入 (Pydantic/Zod)，禁止信任前端。
-  **敏感信息：** 禁止将 Secret/Key 硬编码，必须使用 `.env`。

## 8. 通用设计模式 (Design Patterns)

### 8.1 异步任务 + 轮询模式

**适用场景：** 所有耗时操作（AI推理、文件处理、数据转换）

**核心思想：** 将同步阻塞操作转化为可追踪的异步任务

```
前端 ──POST /task──▶ 后端(返回task_id) ──启动后台线程──▶
                        ▲
前端 ◀──轮询状态───────┘
```

**超时策略：**

| 任务类型 | 轮询间隔 | 超时限制 | 原因 |
|---------|---------|---------|------|
| 短任务 | 1-2s | 60s | 快速反馈 |
| 中等任务 | 3-5s | 10min | 标准配置 |
| 长任务 (AI/ML) | 5-10s | 无限制 | 模型推理不可预估 |

---

### 8.2 共享状态字典模式

**适用场景：** 多线程/异步环境下的状态同步

**核心思想：** 全局字典作为状态中心，后台线程定时写入，前端按需读取

```python
_progress = {
    "task_001": {
        "progress": 45,
        "message": "正在处理...",
        "elapsed": 30,      # 必须写入
        "remaining": 25,     # 必须写入
        "result": None,
        "error": None
    }
}

# 关键：daemon=True + 定时 update()
thread = threading.Thread(target=_progress_updater, daemon=True)
thread.start()
```

**关键要点：**
- 字典字段必须完整，即使某个值没变化也要写入
- 使用 `daemon=True` 确保线程随主进程结束
- 包含 `elapsed` 和 `remaining`，前端才能显示时间

---

### 8.3 批量处理 + 失败回退模式

**适用场景：** AI 批量处理（文本修正、数据标注、内容生成）

**核心思想：** 逐项处理，单项失败不影响整体，结果附带分析说明

```python
for i, item in enumerate(items):
    try:
        result = process_item(item, context=full_list)
        results.append(result)
    except Exception as e:
        # 单项失败不影响整体，回退到原文
        results.append({
            "original": item,
            "corrected": item,
            "analysis": f"处理失败: {e}"
        })
```

---

### 8.4 上下文增强模式

**适用场景：** AI 单项处理需要理解整体语境

**核心思想：** 处理单项时，传入相关上下文，让AI理解完整语义

```
普通做法：处理第3句 → 只看第3句 → 可能误解同音字
增强做法：处理第3句 → 传入 [第2句, 第3句, 第4句] → 上下文准确判断
```

**典型应用：**
- 语音转文字的同音字修正（的/地/得、不/别）
- 口语转书面语的规范化
- 消除歧义的内容处理

---

### 8.5 状态驱动 UI 模式

**适用场景：** 所有前端状态管理场景

**核心思想：** 单一状态对象 + 变更触发重渲染

```javascript
class App {
    state = {
        items: [],
        selectedIds: new Set(),
        status: {},
        isProcessing: false,
        elapsed: 0,
        remaining: 0
    };
    
    setState(updates) {
        Object.assign(this.state, updates);
        this.render();
    }
}
```

### 8.6 状态图标系统模式

**适用场景：** 需要可视化展示任务状态的应用

**核心思想：** 单一图标 + 状态切换 + CSS动画

```
idle → thinking → working → happy
  │         │          │        │
  └─────────┴──────────┴────────┴──→ error
```
